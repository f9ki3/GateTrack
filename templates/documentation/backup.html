GateTrack API Documentation

RESTful API for mobile attendance tracking and RFID gate access.

Flask SQLite JWT RFID
Mobile App

React Native app uses /api/v1/mobile/* endpoints with JWT Bearer auth.

  • Login & Profile Management
  • Attendance History Viewing
  • CSV Data Export
ESP32 RFID

Public endpoints for physical RFID scanners (rfid_scanner).

  • Open Door (Mode 1)
  • Time In (Mode 2)
  • Time Out (Mode 3)

Authentication

Mobile Login Sequence
POST /api/v1/mobile/login
POST /api/v1/mobile/login Content-Type: application/json { "email": "user@example.com", "password": "password123" }
Response (200 OK):
{ "success": true, "token": "eyJhbGciOiJIUzI1NiIs...", "user": {...} }
Protected Routes

Include this header in all protected mobile API calls:

Authorization: Bearer <token>

JWT Secret

gatetrack-jwt-mobile-secret-super-secure-2024-change-in-prod

Endpoints Reference

Endpoint Method Auth Description Parameters Response
/api/v1/mobile/login POST None Login & get JWT JSON: email, password 200: {token, user}
/api/v1/mobile/profile GET
PUT 		JWT Get/Update profile PUT: first_name, last_name, email, contact 200: {user}
/api/v1/mobile/password PUT JWT Change password current_password, new_password, confirm 200: {success: true}
/api/v1/mobile/attendance GET JWT Paginated attendance page, per_page, search, date_from, date_to 200: {page, records: [...]}
/api/v1/mobile/attendance/export/csv GET JWT CSV export search, date_from, date_to CSV File Download
/api/check_users POST None Verify RFID (ESP32) JSON: rfid {status: "success/denied", user}
/api/attendance POST None RFID Log (ESP32) JSON: rfid, mode (1=door, 2=in, 3=out) {status, type, commands}

cURL Examples

Mobile Login
curl -X POST http://192.168.1.100:5000/api/v1/mobile/login \ -H "Content-Type: application/json" \ -d '{"email":"user@example.com","password":"pass123"}'
Protected Request (Profile)
curl -X GET http://192.168.1.100:5000/api/v1/mobile/profile \ -H "Authorization: Bearer YOUR_JWT_TOKEN"
ESP32 RFID Time-In Execution
curl -X POST http://192.168.1.100:5000/api/attendance \ -H "Content-Type: application/json" \ -d '{"rfid":"ABC123DEF456","mode":2}'

Error Handling

401 Unauthorized
{"success": false, "message": "Invalid token"}
400 Bad Request
{"success": false, "message": "Missing RFID parameter"}
404 Not Found
{"success": false, "message": "User not found"}
